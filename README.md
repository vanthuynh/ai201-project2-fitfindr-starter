## Tool Inventory

### Tool 1: search_listings

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
Search mock listing dataset from data/listings.json for matching items. Filter for size then max_price. Return ranked by relevance (best first). Pure/deterministic — no LLM, never raises.


**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `description` (str): keywords used to identify the desired item. (e.g., `"vintage graphic tee"`)
- `size` (str | None): A case-insensitive text filter for sizing that utilizes substring matching. e.g. "M" will successfully match a listing sized "S/M". Pass `None` to bypass this filter entirely
- `max_price` (float | None): maximum price ceiling. Pass `None` to skip filtering

**What it returns:**
<!-- Describe the return value — what fields does a result contain? -->
Format: list of dictionaries`list[dict]`, sorted by highest score, then lowest price.

Fields included: `id`, `title`, `description`, `category`, `style_tags`, `size`, `condition`, `price`, `colors`, `brand`, `platform`.

Scoring: Lowercase keyword substring match. Weights: `style_tags` (3x), `title` (2x), description/brand/category (1x). Items scoring 0 are dropped.

**What happens if it fails or returns nothing:**
<!-- What should the agent do if no listings match? -->

Never raises errors. Returns `[]` on zero matches. The Planning Loop handles recovery retries by first loosening `max_price`, then `size`.

### Tool 2: suggest_outfit

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
Ask LLM (Groq meta-llama/llama-4-scout-17b-16e-instruct) to generate 1-2 complete outfits combining a new listing with the user's existing wardrobe.


**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `new_item` (dict): The thrifted item being considered.
- `wardrobe` (dict): Structured as `{"items": list[dict]}` where items contain `id`, `name`, `category`, `colors`, `style_tags`, and `notes`. Can be empty.

**What it returns:**
<!-- Describe the return value -->
Format: Non-empty `str` containing styling advice.

Populated Wardrobe: Prompts LLM with formatted wardrobe pieces `name (category; style_tags)` to build outfits referencing specific items.

Empty Wardrobe: Prompts for general styling advice. Fabricating fake wardrobe pieces is forbidden.


**What happens if it fails or returns nothing:**
<!-- What should the agent do if the wardrobe is empty or no outfit can be suggested? -->
LLM calls use a `call_llm` helper with a 3x exponential backoff. Total failure returns a graceful error string containing the HTTP status code (e.g., "Groq error 429"). Never raises exceptions or returns `""`.

### Tool 3: create_fit_card

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
 Generates a short, shareable description of a complete outfit — the kind of thing someone would caption an Instagram post with. Runs at a high LLM temperature to ensure varied outputs.


**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `outfit` (...): The styling suggestion generated by `suggest_outfit`.
- `new_item` (dict): The listing data (specifically requires name, price, and platform).

**What it returns:**
<!-- Describe the return value -->
Format: 2-4 sentence `str`.

Requirements: Must capture the outfit's specific vibe while mentioning the item's name, price, and platform exactly once each.

**What happens if it fails or returns nothing:**
<!-- What should the agent do if the outfit data is incomplete? -->
Input Guard: If `outfit` is empty or whitespace, it skips the LLM call entirely and returns a static error string. Does not raise.

LLM Failure: Utilizes the identical `call_llm` backoff and graceful error string behavior as Tool 2.


## Planning Loop Explanation

**Overview**
The `run_agent(query, wardrobe)` function in `agent.py` manages a dynamic execution pipeline. It strictly manages state via a single dictionary and does **not** execute all tools unconditionally.

**Execution Steps**

1. **Initialize State:** Creates a new `session` dictionary (`_new_session`) to act as the single source of truth for the run.
2. **Parse Query:** Attempts to extract `{description, size, max_price}` strictly as JSON via the LLM. If the JSON is invalid or empty, it falls back to a deterministic regex parser. Never raises exceptions. $\rightarrow$ Stores in `session["parsed"]`.
3. **Search & Retry:** Executes the search with parsed filters. If results are empty, it climbs a retry ladder: drops `max_price`, then drops `size`. Records exactly what was loosened in `session["notice"]`.
4. **Evaluate Branch (Early Exit):** If the retry ladder still yields zero matches, it populates `session["error"]` and **returns immediately**. The `suggest_outfit` and `create_fit_card` tools are explicitly skipped to prevent execution on empty inputs.
5. **Select Item:** Grabs the highest-ranked match (`results[0]`) $\rightarrow$ Stores in `session["selected_item"]`.
6. **Suggest Outfit:** Executes styling logic $\rightarrow$ Stores in `session["outfit_suggestion"]`.
7. **Generate Fit Card:** Executes caption logic $\rightarrow$ Stores in `session["fit_card"]`. Returns the final completed `session` dictionary.

## State Management Approach

**Overview**
A single `session` dict (initialized via `_new_session`) serves as the strict single source of truth for each run. Every step writes its output directly to this state before the next step reads it. No hardcode or value reentered by users. `session` doesn't persist between user queries.

**State Schema**

| Key | Type | Written By | Read By |
| :--- | :--- | :--- | :--- |
| `query` | `object` | Entry point | Parser |
| `parsed` | `{description, size, max_price}` | Parse step | Search step |
| `search_results` | `list[dict]` | Search / Retry ladder | Selection step |
| `notice` | `str` | None` *(Added field)* | Retry ladder | UI |
| `selected_item` | `dict` | Selection step | `suggest_outfit` / UI |
| `wardrobe` | `dict` | Entry point | `suggest_outfit` |
| `outfit_suggestion` | `str` | `suggest_outfit` | `create_fit_card` / UI |
| `fit_card` | `str` | `create_fit_card` | UI |
| `error` | `str` | None` | Any early-exit branch | UI *(checked first)* |
